# Propolys 개발 프로젝트

3D 공간을 탐색하며 퍼즐을 해결하는 싱글 플레이어 게임 프로젝트.

Unity 기반으로 제작되었으며, 씬 전환 간의 데이터 보존과 안정적인 게임 루프, 효율적인 UI 전환 시스템, 그리고 확장성 높은 상호작용 및 정밀한 물리 장치 시스템 구축을 목표로 개발을 진행했습니다.

---

# 게임 소개

플레이어는 3D 공간에서 시점을 회전하고 이동하며 배치된 다양한 퍼즐 오브젝트들과 상호작용합니다.

스테이지에 구현된 물리 연산과 장치들을 활용하여 정해진 목표를 완수하고 퍼즐을 해결하는 것을 목표로 합니다.

# 핵심 시스템

- **중앙 집중형 매니저 구조 (MainManager & PauseManager)**
  - 오디오, 마우스 감도 등 글로벌 데이터와 일시정지 로직을 분리하여 시스템 간 결합도를 낮춤
- **상태 기반 UI 및 마우스 커서 제어**
  - 인게임 HUD와 일시정지 패널 간의 상태 전환(`SetActive`) 자동화
  - 1인칭/3인칭 시점에 맞춘 마우스 커서 숨김/고정(`CursorLockMode`) 및 해제 기능 구현
- **동적 오디오 믹서(Audio Mixer) 연동**
  - BGM 및 SFX 볼륨 슬라이더 수치를 수학적 로그 스케일($\text{dB} = \log_{10}(\text{value}) \times 20$)로 변환하여 자연스러운 음량 조절 기능 구현
- **현실 시간 기반(Realtime) 예외 처리 코루틴**
  - `Time.timeScale = 0f`를 활용한 시간 정지 상태에서도 씬 전환 및 연출 지연 시간이 정상 작동하도록 `WaitForSecondsRealtime`을 반영한 페이드/딜레이 시스템
- **동적 이벤트 바인딩 시스템**
  - 씬 전환 시 기존 UI가 파괴되더라도, 새 씬의 슬라이더와 버튼들이 씬 내부 매니저를 자동으로 추적(`FindFirstObjectByType`)하여 설정값을 동기화하는 구조 설계
- **인터페이스 기반 상호작용 및 동적 장치(엘리베이터) 제어 시스템**
  - 객체지향 설계인 **인터페이스(Interface)**를 활용하여 버튼, 상자, 문 등 다양한 상호작용 오브젝트의 무한한 기능 확장이 가능한 구조 구축
  - **카메라 레이캐스트(Raycast)** 기반의 정밀한 오브젝트 타겟팅과 마우스 왼쪽 클릭(`Input.GetMouseButtonDown(0)`) 상호작용 매커니즘 구현
  - **시선 방향(World Direction) 유니티 로컬 좌표계 변환(`InverseTransformDirection`) 기술**을 도입하여, 버튼의 배치 각도와 무관하게 플레이어가 바라보는 시선 방향으로 버튼이 물리적으로 눌렸다 나오는 연출 구현
  - 컴포넌트 간 결합도를 낮추기 위해 **유니티 이벤트(UnityEvent)** 아키텍처를 채택하여, 버튼 컴포넌트 수정 없이 인스펙터 상에서 엘리베이터 문의 열림/닫힘(`OpenDoor`/`CloseDoor`) 기능을 자유롭게 연결 및 바인딩하도록 설계
  - `Mathf.SmoothStep` 보간 함수와 **코루틴(Coroutine)**을 활용하여 실제 엘리베이터처럼 문이 열리고 닫힐 때 가속 및 감속이 일어나는 부드러운 하드웨어 연출 제어
- **물리 컴포넌트 활용 저울 퍼즐 시스템**
  - 유니티 3D 물리 엔진의 **힌지 조인트(Hinge Joint)** 물리 제약 조건을 활용하여 회전축이 고정된 지포형/양팔 저울 구조 설계
  - 강체(**Rigidbody**)의 질량(`Mass`) 및 마찰력 설정을 튜닝하여 오브젝트 무게에 따라 실시간으로 저울의 기울기가 변하는 물리 퍼즐 매커니즘 구현
  - **트리거 충돌 처리(`OnTriggerEnter` / `Exit`)** 및 공간 검사 기법을 통해 특정 구역에 도달한 물리 오브젝트들의 무게 합산을 연산하고, 퍼즐의 정답 조건(양쪽 무게 균형 또는 특정 무게 달성)을 판별하는 감지 로직 구축

# 개발 목표

단순히 기능을 구현하는 것을 넘어 발생할 수 있는 구조적 문제를 예방하는 것에 집중했습니다.

- 씬 전환 시 오브젝트 생명 주기에 영향을 받지 않는 **안정적인 데이터 유지**
- 게임 정지 상태와 물리 정지 상태를 명확히 분리한 **시간 제어 설계**
- 하드코딩된 인펙터 연결을 탈피하여 씬이 바뀌어도 끊기지 않는 **유연한 UI 컴포넌트 구조**
- 스크립트 수정 최소화와 높은 재사용성을 보장하는 **이벤트 및 인터페이스 기반 컨텐츠 확장성 확보**
- 스크립트 연산 위주의 연출을 탈피하고, 유니티 컴포넌트 자체 물리 법칙을 이용한 **신뢰성 높은 물리 퍼즐 환경 구축**

를 목표로 개발을 완료했습니다.

# 개발 기록

프로젝트 진행 중 씬 전환 시 인스펙터 연결이 끊어지는 싱글톤의 한계를 경험한 후, 각 씬의 UI가 매니저를 역으로 추적하여 값을 동기화하는 구조로 리팩토링을 완료했습니다. 일시정지 시 코루틴이 멈추는 데드락 이슈를 해결하며 유니티 시간 시스템에 대한 이해도를 높였습니다. 

또한, 하드코딩 방식으로 상호작용을 구현할 때 스크립트가 비대해지는 문제를 인터페이스(`IInteractable`)와 `UnityEvent` 결합 구조로 리팩토링했습니다. 이를 통해 버튼 클릭 시 플레이어 시선 방향을 계산해 물리적 반동을 주는 코루틴 연출과 엘리베이터 문 제어 시스템을 구현했습니다. 나아가 단순 좌표 이동이 아닌 **힌지 조인트(Hinge Joint)** 등 물리 관절 컴포넌트를 이용해 실제 오브젝트의 질량(`Mass`) 변화에 실시간으로 반응하는 저울 퍼즐 구조를 설계하면서, 유니티 물리 엔진(PhysX)의 제약 조건 처리 방식과 리지드바디 제어 역량을 한층 더 강화했습니다.
